ISSUE 1: Supabase-py v2.16 SDK - Storage URL Construction Broken
================================================================================
What:    client.storage.get_bucket() / create_bucket() constructs invalid URLs
         for bucket management operations. The SDK appends /rest/v1 incorrectly,
         breaking the storage REST path (e.g. /storage/v1/bucket/xxx becomes
         /rest/v1/storage/v1/bucket/xxx).
Where:   accounts/utils/supabase_storage.py (init)
         accounts/management/commands/backup_daily_full.py (bucket check/create)
         accounts/utils/drive_backup_service.py (bucket check)
Why:     supabase-py v2.x changed internal URL routing. The SDK assumes a
         specific supabase-js pattern that doesn't match the Python REST client.
         PyPI package locked at 2.16.0.
Fix:     Replace all bucket CRUD with direct HTTP requests:
           GET  https://{url}/storage/v1/bucket/{name}
           POST https://{url}/storage/v1/bucket
         Also replaced list operations with direct GET /storage/v1/object/list/
         See commit e303d0f.

================================================================================

         accounts/utils/storage_manager.py (upload_to_supabase_storage)

         falls back to content_type/upsert on TypeError.

         See commits a219183, 3647316, 1b91056.

ISSUE 9: Django ASGI - Background Threads Broken in WebSocket Mode
================================================================================
What:    Post_save signals that spawn background threads (threading.Thread)
         to upload files to Google Drive hang or crash in ASGI/Daphne mode.
         The background thread cannot acquire the Django database connection.
Where:   accounts/models.py (post_save signal handlers)
         accounts/utils/backup_trigger.py (backup_signup_pdf_on_save,
         backup_teacher_resource_on_save)
Why:     Daphne runs in ASGI mode. Django's database connections are per-thread
         and ASGI's async event loop doesn't properly support thread spawning
         for DB operations. Threads deadlock waiting for connection.
Fix:     Remove post_save signal handlers entirely. Move all backup logic to
         Supabase Edge Function (drive-backup), triggered by database webhook
         on the storage.objects table INSERT event. Zero Django RAM/threads.
         See commit ad400db, bbe1204.

ISSUE 30: Malware Scanner - PDF False Positives (MZ/ELF Bytes)
================================================================================
What:    Malware scanner flags legitimate PDF files as malicious because PDF
         containers can legitimately contain MZ (Windows executable) or ELF
         (Linux binary) byte sequences within embedded streams.
Where:   accounts/utils/malware_scanner.py (scan_file)
Why:     Byte signature scanning checks for MZ (0x4D5A) and ELF (0x7F454C46)
         magic bytes at any position, but these can appear inside PDF binary
         streams that are part of the legitimate file format.
Fix:     Early-return for PDFs: skip byte signature + entropy checks for PDF
         MIME types. Only check file size (<10MB) and valid PDF header (%PDF-).
         Full scan still runs on images, DOCX, etc. See commit 27723f6.
